  <files>tests/test_extractor.py, tests/test_synthesizer.py</files>
  <action>
Add focused edge case tests for the two main response parsers. These are the most likely real-world failure modes (Claude returns unexpected formats).

**In tests/test_extractor.py**, add a new test class `TestParseExtractionEdgeCases`:

1. `test_empty_response()` — pass empty string "" to `_parse_extraction_response()`. Should return a MeetingExtraction with all empty lists and low_signal=True.

2. `test_no_sections_response()` — pass "Just some random text without any ## headers" to parser. Should return MeetingExtraction with empty lists and low_signal=True.

3. `test_partial_sections()` — pass response with only "## Decisions\n- Something decided | Alice | Reason" (missing other sections). Should parse decisions correctly, other sections empty.

4. `test_none_values_in_sections()` — pass response where a section just says "None" or "None." under the header. Should return empty list for that section.

5. `test_malformed_pipe_delimited()` — pass "## Decisions\n- Missing pipes here no delimiter" to parser. Should still parse as plain bullet item.

**In tests/test_synthesizer.py**, add a new test class `TestParseSynthesisEdgeCases`:

1. `test_empty_response()` — pass empty string "" to `_parse_synthesis_response()`. Should return dict with empty lists and None executive_summary.

2. `test_no_sections_response()` — pass "Random text no headers" to parser. Should return empty result dict.

3. `test_partial_sections()` — pass response with only "## Substance\n- Item one" (missing Decisions/Commitments). Substance should have 1 item, others empty.

4. `test_commitments_table_format()` — pass response with a properly formatted markdown table under "## Commitments". Should parse table rows correctly.

5. `test_no_items_text()` — pass response where all sections contain "No items for this day." Should return empty lists for all sections.

6. `test_missing_executive_summary()` — pass response with Substance/Decisions/Commitments but no Executive Summary section. executive_summary should be None.

Import the parser functions directly (they're already importable as they start with underscore but are in the same package):
- `from src.synthesis.extractor import _parse_extraction_response`
- `from src.synthesis.synthesizer import _parse_synthesis_response`
