2. Create `src/retry.py` with a shared retry decorator:
   - `retry_api_call` decorator using tenacity:
     - `stop=stop_after_attempt(3)` (initial attempt + 2 retries)
     - `wait=wait_exponential(multiplier=1, min=1, max=4)` (1s, 2s, 4s waits)

       - `slack_sdk.errors.SlackApiError` with response status 429 or "ratelimited"
     - `before_sleep` callback that logs at WARNING level: "Retry {attempt}/{max} for {func_name}: {error}"
     - `reraise=True` so final failure propagates the original exception
   - Helper function `is_retryable_google_error(exc)` that checks HttpError status codes
   - Helper function `is_retryable_slack_error(exc)` that checks Slack error codes
   - Import guards: use try/except for optional dependencies (googleapiclient, slack_sdk) so the module works even if they're not installed

<task type="auto">
  <name>Task 3: Add token budget estimation and truncation to synthesizer</name>
  <files>src/synthesis/synthesizer.py</files>
  <action>
Add token budget enforcement to `synthesize_daily()` BEFORE the Claude API call:

1. Add constants at module level:
   ```python
   TOKEN_BUDGET = 100_000
   CHARS_PER_TOKEN = 4
   CHAR_BUDGET = TOKEN_BUDGET * CHARS_PER_TOKEN  # 400,000 chars
   SAFETY_MARGIN = 0.8  # Use 80% of budget for safety
   EFFECTIVE_CHAR_BUDGET = int(CHAR_BUDGET * SAFETY_MARGIN)  # 320,000 chars
   ```

2. Add a `_estimate_and_truncate()` function that:
   - Takes the formatted prompt components: extractions_text, slack_items_text, docs_items_text, hubspot_items_text, and the base prompt template
   - Calculates total character count
   - If under EFFECTIVE_CHAR_BUDGET, returns all components unchanged with truncated_sources=[]
   - If over budget, truncates in priority order (lowest first):
     1. docs_items_text -> "" (remove Docs first)
     2. hubspot_items_text -> "" (remove HubSpot second)
     3. slack_items_text -> "" (remove Slack third)
     4. extractions_text is NEVER truncated (highest priority — meetings/transcripts)
   - After each truncation, re-check if under budget
   - Returns the (possibly truncated) components and a list of truncated source names
   - Log at WARNING level which sources were truncated

3. In `synthesize_daily()`, call `_estimate_and_truncate()` after formatting but before building the final prompt. If any sources were truncated, include a warning note in the returned dict:
   ```python
   result["truncated_sources"] = truncated_sources  # e.g., ["Google Docs", "HubSpot"]
   ```

4. Update the pipeline runner (`src/pipeline.py` is NOT in this plan's files — the truncated_sources info flows through the dict return and will be consumed by the writer in a future plan or as a log warning). For now, just log the warning in synthesizer.

5. Add tests in a new section at the bottom of `tests/test_synthesizer.py`:
   - Test `_estimate_and_truncate()` with content under budget (no truncation)
   - Test with content over budget that truncates docs first, then hubspot
   - Test that extractions are never truncated even when total is way over budget
   - Test the truncated_sources list is populated correctly
  </action>
