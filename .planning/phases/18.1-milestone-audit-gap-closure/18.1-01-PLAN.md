Fix the str->Path bug in `run_notion_discovery()` at line 95 of `src/ingest/notion_discovery.py`.

**The bug:** `run_notion_discovery(config_path: str)` passes raw string to `load_config()` which expects `Path | None`. When `load_config` calls path operations like `.exists()` on the string, it crashes with `AttributeError`.

**The fix:** On line 95, change:
```python
config = load_config(config_path)
```
to:
```python
config = load_config(Path(config_path))
```

Add `from pathlib import Path` import if not already present at the top of the file.

**Add a regression test** in `tests/test_notion_discovery.py`:

- Name the test `test_run_notion_discovery_passes_path_to_load_config`

All other callers of `load_config` already pass `Path` objects correctly -- this is the only broken call site.
