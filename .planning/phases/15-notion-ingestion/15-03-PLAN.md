Add a Notion Activity section AFTER the HubSpot section (before the Per-Meeting Extractions). Follow the HubSpot section pattern with grouping:

```jinja2
{% if notion_items %}
## Notion Activity

{% set page_items = notion_items | selectattr("source_type.value", "equalto", "notion_page") | list %}
{% set db_items = notion_items | selectattr("source_type.value", "equalto", "notion_db") | list %}
{% if page_items %}
### Pages Edited ({{ page_items | length }})

{% for item in page_items %}
- **{{ item.title }}** — {{ item.content[:200] }}{% if item.content|length > 200 %}...{% endif %} {{ item.attribution_text() }}
{% endfor %}

{% endif %}
{% if db_items %}
{# Group DB items by display_context (which contains database name) #}
{% set db_groups = {} %}
{% for item in db_items %}
{% set group = item.display_context %}
{% if group not in db_groups %}
{% set _ = db_groups.update({group: []}) %}
{% endif %}
{% set _ = db_groups[group].append(item) %}
{% endfor %}
{% for group_name, items in db_groups.items() %}
### {{ group_name }} ({{ items | length }})

{% for item in items %}
- **{{ item.title }}** — {{ item.content[:200] }}{% if item.content|length > 200 %}...{% endif %} {{ item.attribution_text() }}
{% endfor %}

{% endfor %}
{% endif %}
{% endif %}
```

Note: Jinja2 dict operations can be tricky. A simpler alternative for grouping:

```jinja2
{% if db_items %}
{% set ns = namespace(seen=[]) %}
{% for item in db_items %}
{% if item.display_context not in ns.seen %}
{% set ns.seen = ns.seen + [item.display_context] %}
### {{ item.display_context }} Changes

{% for inner_item in db_items %}
{% if inner_item.display_context == item.display_context %}
- **{{ inner_item.title }}** — {{ inner_item.content[:200] }}{% if inner_item.content|length > 200 %}...{% endif %} {{ inner_item.attribution_text() }}
{% endif %}
{% endfor %}

{% endif %}
{% endfor %}
{% endif %}
```

Use whichever approach works cleanly in the template. The key requirement: DB items grouped by database name (from display_context).
