  <action>
Create src/ingest/notion_discovery.py following the slack_discovery.py interactive pattern.

**Functions:**

**_scan_databases(client: NotionClient) -> list[dict]**:
- Use POST /search with filter {"property": "object", "value": "database"}
- Paginate through all results
- For each database, extract: id, title (from title rich_text), description (if present), last_edited_time
- Sort by last_edited_time descending (most recently active first)
- Return list of dicts with: id, title, description, last_edited_time

**_get_database_item_count(client: NotionClient, db_id: str) -> int**:
- POST /databases/{db_id}/query with empty filter, page_size=1
- Return the total from response (or estimate from has_more)
- Note: Notion doesn't return total count. We can either:
  - Just report "has items" / "empty" based on results length
  - Or paginate to count (too expensive). Go with the simpler approach.

**run_notion_discovery(config_path: str) -> None** (main entry point):
1. Load config from config_path using load_config()
2. Check config.notion.token is set. If not, print error and exit:
   "No Notion token configured. Add notion.token to config.yaml first."
3. Build NotionClient from token
4. Scan databases
5. If no databases found, print "No databases found. Make sure databases are shared with your integration." and return
6. Display table:
   ```
   Found {N} Notion databases:

     #  Database                         Last edited
     1  Project Tasks                    2026-04-05
     2  Bug Tracker                      2026-04-04
     3  Content Calendar                 2026-04-01
   ```
7. Show currently watched databases (from config.notion.watched_databases) with checkmarks
8. Prompt: "Enter numbers to watch (comma-separated), or 'done' to finish: "
9. Parse user input, map to database IDs
10. Update config.yaml:
    - Read existing YAML
    - Set notion.watched_databases to selected database IDs
    - Write back (use yaml.dump with default_flow_style=False)
    - Print confirmation: "Updated config.yaml with {N} watched databases."
