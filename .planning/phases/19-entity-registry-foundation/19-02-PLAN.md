1. **src/entity/repository.py** -- Data access layer. ALL sqlite3 imports confined here and db.py only (per user decision). Follow project conventions: `from __future__ import annotations`, Google-style docstrings, `logger = logging.getLogger(__name__)`.

`EntityRepository` class:
- `__init__(self, db_path: str)` -- stores db_path, sets `_conn = None`
- `connect(self) -> None` -- calls `get_connection(self._db_path)` from db.py, stores in `_conn`
- `close(self) -> None` -- closes connection if open
- `__enter__` / `__exit__` -- context manager support (connect on enter, close on exit)

Entity CRUD:
- `add_entity(self, name: str, entity_type: str, organization_id: str | None = None, hubspot_id: str | None = None, slack_channel: str | None = None, metadata: dict | None = None) -> Entity` -- generates uuid4().hex id, generates timestamps via `_now_utc()` from models.py, INSERTs into entities table, returns Entity model. Raise ValueError if name is empty.
- `get_by_id(self, entity_id: str) -> Entity | None` -- SELECT with `deleted_at IS NULL` filter.
- `get_by_name(self, name: str) -> Entity | None` -- case-insensitive lookup by canonical name (`LOWER(name) = LOWER(?)`) with `deleted_at IS NULL`.
- `list_entities(self, entity_type: str | None = None, include_deleted: bool = False) -> list[Entity]` -- SELECT with optional type filter, ordered by name. Adds `deleted_at IS NULL` unless include_deleted=True.
- `remove_entity(self, entity_id: str) -> bool` -- soft-delete: sets deleted_at to current timestamp. Returns True if entity was found and soft-deleted, False if not found.
- `_row_to_entity(self, row: sqlite3.Row) -> Entity` -- converts Row to Entity model, handling metadata JSON deserialization.

Alias management:
- `add_alias(self, entity_id: str, alias: str) -> Alias` -- generates uuid4().hex id, INSERTs into aliases table. Raise ValueError if alias already exists (UNIQUE constraint). Raise ValueError if entity_id does not exist.
- `remove_alias(self, alias: str) -> bool` -- DELETE from aliases where alias matches (case-insensitive: `LOWER(alias) = LOWER(?)`). Returns True if deleted.
- `list_aliases(self, entity_id: str) -> list[Alias]` -- SELECT aliases for entity.
- `resolve_name(self, name: str) -> Entity | None` -- two-step resolution per research: (1) try canonical name via get_by_name, (2) try alias lookup with case-insensitive match (`LOWER(a.alias) = LOWER(?)`). Returns Entity or None. If the resolved entity has a merge_target_id, follow the pointer (one level only -- no recursive chain).

All methods use `self._conn` directly. Use `%s` logging format (not f-strings). Use standard SQL (no SQLite-specific features per user decision about Postgres compatibility). Generate all timestamps in Python via the `_now_utc()` helper.

1. **src/entity/cli.py** -- CLI subparser registration and command handlers. Follow existing project pattern from discover-slack/discover-notion.

`register_entity_parser(subparsers) -> None`:
Register `entity` as a subparser with its own sub-subparsers for commands:
- `entity add <name> --type partner|person [--org <org_name>] [--json]` -- create entity
- `entity list [--type partner|person] [--json]` -- list entities
- `entity show <name> [--json]` -- show single entity with aliases
- `entity remove <name>` -- soft-delete entity
- `entity alias add <entity_name> <alias>` -- add alias
- `entity alias list <entity_name>` -- list aliases
- `entity alias remove <alias>` -- remove alias

`handle_entity_command(args: argparse.Namespace) -> None`:
Dispatch based on `args.entity_command`. Load config via `load_config(Path(args.config))`. Create EntityRepository from config.entity.db_path. Use context manager for repo.

Output formatting:
- Default: human-readable table using simple formatted strings (no external dependency). For `entity list`: columns for Name, Type, Created, with header and separator line. For `entity show`: entity details + alias list.
- `--json` flag: output `entity.model_dump_json(indent=2)` or JSON array for lists.

Error handling: catch ValueError from repository (e.g., duplicate alias, empty name) and print user-friendly message to stderr with sys.exit(1). Catch repository connection errors and print "Entity database unavailable" message.
