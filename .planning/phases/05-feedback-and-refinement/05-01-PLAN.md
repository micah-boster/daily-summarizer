  2. Create `src/priorities.py` with:
     - `PriorityConfig` Pydantic model with four list[str] fields: projects, people, topics, suppress (all default to empty lists)
     - `load_priorities(config_path: Path | None = None) -> PriorityConfig` — loads from config/priorities.yaml, returns empty PriorityConfig if file missing (graceful degradation)
     - `build_priority_context(priorities: PriorityConfig, extractions: list[MeetingExtraction]) -> str` — scans extractions and builds a prompt injection string:
       - For each priority category, check if any extraction content matches (substring, case-insensitive)
       - For suppress list, match against meeting_title
       - Return formatted string like:
         ```
         PRIORITY CONTEXT:
         High-priority projects detected: Project Alpha (in "Weekly Sync", "Planning Session")
         High-priority people detected: Sarah Chen (in "Weekly Sync")
         High-priority topics detected: hiring (in "Recruiting Update")
         Suppress: Daily Standup, All Hands (minimize to one-liner each)

         PRIORITY INSTRUCTIONS:
         - Give priority-matched items dedicated subsections with expanded detail
         - Give suppress-matched meetings one-liner treatment: "Also: [title] — [one-sentence summary]"
         - Do NOT add [priority] tags or markers — shape output naturally
         - Non-priority items still appear with standard treatment
         ```
       - If no matches found for any category, omit that category from the output
       - If PriorityConfig is completely empty (no items in any list), return empty string

  3. Create `tests/test_priorities.py` with:
     - Test loading valid priorities.yaml

     - Test build_priority_context with matching extractions produces correct prompt text
     - Test build_priority_context with no matches returns empty string
     - Test build_priority_context with suppress list matches
     - Test case-insensitive matching

  4. Modify `src/synthesis/synthesizer.py`:

<task type="auto">
  <name>Task 2: Diff-based quality metrics tracking</name>
  <files>src/quality.py, src/main.py, tests/test_quality.py</files>
  <action>
  1. Create `src/quality.py` with:
     - `save_raw_output(content: str, target_date: date, output_dir: Path) -> Path` — saves raw pipeline output to `output/raw/daily/YYYY/MM/YYYY-MM-DD.raw.md`. Creates directories as needed. Returns path.
     - `detect_edits(target_date: date, output_dir: Path) -> dict | None` — compares `output/raw/daily/YYYY/MM/YYYY-MM-DD.raw.md` against `output/daily/YYYY/MM/YYYY-MM-DD.md`:
       - Uses `difflib.SequenceMatcher` for similarity ratio
       - Uses `difflib.unified_diff` for line-level changes
       - Parses diff hunks to attribute changes to sections (Substance, Decisions, Commitments, other)
       - Returns dict: `{"date": str, "edited": bool, "similarity": float, "sections_changed": list[str], "additions": int, "deletions": int}` or None if either file missing
     - `update_quality_report(edit_result: dict, output_dir: Path) -> Path`:
       - Appends edit_result as JSON line to `output/quality/metrics.jsonl` (one JSON object per line, append mode)
       - Regenerates `output/quality/quality-report.md` from all metrics.jsonl entries:
         - Last 7 days table: Date, Edit Detected, Similarity, Sections Changed
         - Trends section: 7-day edit rate, 30-day edit rate, most-edited section, average similarity
         - Data volume section: average meetings/day, average transcripts/day (from future integration — placeholder for now)
       - Returns path to quality-report.md
     - Use only stdlib: `difflib`, `json`, `pathlib`, `datetime`

  2. Modify `src/main.py` `run_daily()`:
     - Import `save_raw_output, detect_edits, update_quality_report` from `src.quality`
     - BEFORE calling `write_daily_summary()`: run `detect_edits()` for the current date to check if yesterday's raw output was edited (or the current date if re-running). If edit_result is not None, call `update_quality_report(edit_result, output_dir)` and log the result.
     - AFTER calling `write_daily_summary()`: read the written file content back, call `save_raw_output(content, current, output_dir)` to save the raw copy for future comparison.
     - Wrap quality tracking in try/except — quality tracking failure must not block the pipeline. Log warning on failure.

  3. Create `tests/test_quality.py` with:

     - Test detect_edits with missing files returns None
     - Test update_quality_report appends to metrics.jsonl
     - Test update_quality_report generates quality-report.md with correct structure
     - Test quality report trends calculation with multiple entries
