    - `load_credentials() -> Credentials | None`: Load from token.json, auto-refresh if expired, save refreshed token, return None if refresh fails

IMPORTANT: Pass `access_type='offline'` and `prompt='consent'` to ensure refresh token is always returned (per Pitfall 4 from research). Save credentials after flow completes.

    Follow 00-RESEARCH.md Pattern 2. Implement:
    - Load `SLACK_WEBHOOK_URL` from environment (use `os.environ.get`)
    - Do NOT add python-dotenv as dependency. Keep it simple: environment variable only.
    - `notify_slack(message: str) -> bool`: POST to webhook URL with `{"text": message}`, timeout 10s, return True/False. Log to stderr if webhook URL not set (don't raise).

    **Step 4: Run log module** (`src/validation/run_log.py`)
    Follow 00-RESEARCH.md JSONL pattern. Implement:
    - `VALIDATION_LOG = Path("output/validation/validation_log.jsonl")`
    - `append_to_log(entry: dict) -> None`: Create parent dirs, append JSON line
    - `count_passes() -> int`: Read JSONL, count entries with `status == "pass"`
    - `get_recent_entries(n: int = 10) -> list[dict]`: Return last N log entries for quick review

    **Step 5: Main validation script** (`src/validation/daily_check.py`)
    Follow 00-RESEARCH.md Pattern 3. Implement:
    - `RETRY_DELAY_SECONDS = 15 * 60`
    - `fetch_todays_events(creds) -> list[dict]`: Use `googleapiclient.discovery.build("calendar", "v3", credentials=creds)`. Fetch today's events in US/Eastern timezone (use `zoneinfo.ZoneInfo("America/New_York")`). Return list of dicts with `title`, `start`, `end`, `attendees` (email list), `duration_minutes`.
    - `run_validation() -> dict`: Load credentials (fail with RuntimeError if None, triggering Slack notification for re-auth), call `fetch_todays_events()`, return result dict with `timestamp` (UTC ISO), `status` ("pass"), `event_count`, `events` list.
    - `write_output(result: dict) -> Path`: Write to `output/validation/YYYY-MM-DD_HH-MM.json` (UTC timestamp in filename). Create parent dirs. Return path.
    - `main()`: Entry point with retry logic per locked decision. On first failure: log error + notify Slack + sleep 15 min + retry. On second failure: log + notify + return. On success: write output + append to log + notify Slack with event count + print pass count from log.
    - Add `if __name__ == "__main__"` block calling `main()`

    **Important implementation notes:**
    - Use `google-auth` library (NOT deprecated `oauth2client`) per research State of the Art
    - Use `zoneinfo` from stdlib (NOT `pytz`) per research State of the Art
    - All file I/O uses `pathlib.Path` for consistency
    - The validation script is a single-shot execution (NOT a daemon/scheduler) per anti-pattern guidance -- Cowork handles scheduling
    - Error messages in Slack notifications should include enough context for the user to diagnose without checking logs (include error type, brief message, suggestion like "run re-auth procedure")
