<objective>
Fix the broken command palette date navigation (NAV-05 partial gap).

Currently, selecting a date in the Cmd+K palette calls `addRecentDate` and `setActiveTab("summaries")` but has no path to `page.tsx`'s local `selectedDate` state. The center panel stays on whatever date was previously selected.

The fix: lift `selectedDate` and `setSelectedDate` into the Zustand `ui-store.ts` so that both `page.tsx` and `command-palette.tsx` share the same state. The command palette's `handleSelectDate` calls `setSelectedDate(dateStr)` and `page.tsx` reads it from the store.

Purpose: Close the integration gap identified in the v1.0 milestone audit -- command palette date selection must actually navigate the center panel.
Output: Working Cmd+K date navigation that switches the center panel to the selected date's summary.
</objective>
