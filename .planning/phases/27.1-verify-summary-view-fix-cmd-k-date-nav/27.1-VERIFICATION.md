## NAV-05 Fix Verification (Critical Path)

The NAV-05 gap was that `command-palette.tsx` had no path to `page.tsx`'s `selectedDate`. Plan 27.1-01 fixed this by lifting both `selectedDate` and `selectedViewType` out of local `useState` into the Zustand `ui-store.ts`.

Full chain verified:
1. User types date in Cmd+K
2. `parseNaturalDate` parses input
3. `handleSelectDate(dateStr)` called
4. `addRecentDate(dateStr)` -- updates recent dates list in store
5. `setActiveTab("summaries")` -- switches tab if on entities
6. `setSelectedDate(dateStr)` -- stores in Zustand (the fix)
7. `setSelectedViewType("daily")` -- ensures daily view mode (the fix)
8. `page.tsx` reads `selectedDate` from store (line 39) -- reacts to change
9. `selectedType === "daily"` is true -> `<SummaryView selectedDate={selectedDate} />` renders

No local `useState` for `selectedDate` remains in page.tsx. Confirmed: no `useState.*selectedDate` in page.tsx.

Commits verified: `6f5037d` (lift state) and `6b44e65` (wire palette) both exist in git log.
