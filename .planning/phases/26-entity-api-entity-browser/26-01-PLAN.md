  <action>
Add a `get_related_entities(self, entity_id: str, limit: int = 10) -> list[dict]` method to EntityRepository.

This method queries entity_mentions to find other entities that co-occur with the given entity on the same source_date AND source_id. Returns a list of dicts with keys: entity_id, name, entity_type, co_mention_count. Sorted by co_mention_count descending, limited to `limit` results. Excludes the entity itself and soft-deleted entities.

SQL approach:
```sql
SELECT e.id AS entity_id, e.name, e.entity_type, COUNT(*) AS co_mention_count
FROM entity_mentions em1
JOIN entity_mentions em2 ON em1.source_date = em2.source_date
    AND em1.source_id = em2.source_id
    AND em1.entity_id != em2.entity_id
JOIN entities e ON em2.entity_id = e.id AND e.deleted_at IS NULL
WHERE em1.entity_id = ?
GROUP BY e.id, e.name, e.entity_type
ORDER BY co_mention_count DESC
LIMIT ?
```

Also include mentions from merged entities (where merge_target_id = entity_id) in the em1 side, same as get_entity_mentions_in_range does.
