  <files>src/ingest/gmail.py, config/config.yaml</files>
  <action>
Create `src/ingest/gmail.py` with the following functions, mirroring the pattern from `calendar.py`:

1. `build_gmail_service(creds: Credentials)` — Returns `build("gmail", "v1", credentials=creds)`. Same pattern as `build_calendar_service`.

2. `search_messages(service, query: str, max_results: int = 100) -> list[dict]` — Call `service.users().messages().list(userId="me", q=query, maxResults=min(max_results, 500))`, handle pagination via `nextPageToken`, return list of message stubs (each has `id` and `threadId`).

3. `get_message_content(service, msg_id: str) -> dict` — Call `service.users().messages().get(userId="me", id=msg_id, format="full")`. Return the full message dict.

4. `extract_headers(message: dict) -> dict[str, str]` — From `message["payload"]["headers"]`, extract and return a dict with keys: `subject`, `from`, `to`, `date`. Case-insensitive header name matching.

5. `extract_body_text(message: dict) -> str` — Extract plain text from the message payload. Handle three cases:
   - Single-part: decode `payload["body"]["data"]` using `base64.urlsafe_b64decode`
   - Multipart: recursively walk `payload["parts"]`, prefer `text/plain`, fallback to `text/html` with HTML tag stripping
   - Use `html.unescape()` for HTML entities
   Return empty string if no body found.

6. `build_transcript_query(sender_patterns: list[str], subject_patterns: list[str], target_date: date) -> str` — Build Gmail search query string. Format: `({from:sender1 OR from:sender2}) ({subject:kw1 OR subject:kw2}) after:YYYY/MM/DD before:YYYY/MM/DD`. Note: Gmail date format uses slashes, not dashes.

7. `cache_raw_emails(emails: list[dict], source_name: str, target_date: date, output_dir: Path) -> Path` — Cache raw email data to `output_dir/raw/YYYY/MM/DD/{source_name}_emails.json`. Follow the same directory structure as `cache_raw_response` in calendar.py.

<task type="auto">
  <name>Task 2: Gemini transcript parser, filler stripping, and tests</name>
  <files>src/ingest/transcripts.py, tests/test_gmail_ingest.py</files>
  <action>
Create `src/ingest/transcripts.py` with:

1. `FILLER_PATTERNS: list[str]` — Compiled regex patterns for filler removal:
   - `r'\b(?:um|uh|ah|er|mm|hmm|hm)\b'` — common fillers
   - `r'\b(?:like,?\s+)?you know,?\s*'` — "like, you know"
   - `r'\b(?:I mean,?\s*)'` — "I mean"
   - `r'\b(\w+)\s+\1\b'` — repeated words ("the the" -> "the")

2. `strip_filler(text: str) -> str` — Apply all filler patterns with `re.sub(pattern, ' ', text, flags=re.IGNORECASE)`, then collapse multiple spaces with `re.sub(r'\s{2,}', ' ', text).strip()`. Return cleaned text.

3. `parse_gemini_transcript(message: dict, config: dict) -> dict | None` — Parse a Gmail message into a transcript dict:
   - Extract headers using `extract_headers` from gmail.py
   - Extract body text using `extract_body_text` from gmail.py
   - If body is empty, return None
   - Parse meeting title from subject line (strip common prefixes like "Transcript for ", "Meeting notes: ", "Notes from ")
   - Parse meeting datetime from email Date header using `dateutil.parser.parse`
   - If config `transcripts.preprocessing.strip_filler` is true, apply `strip_filler` to body
   - Return dict: `{"source": "gemini", "title": str, "meeting_time": datetime, "transcript_text": str, "raw_email": dict, "message_id": str}`

4. `fetch_gemini_transcripts(service, target_date: date, config: dict) -> list[dict]` — Orchestrator function:
   - Read sender_patterns and subject_patterns from `config["transcripts"]["gemini"]`
   - Build query using `build_transcript_query`
   - Search messages using `search_messages`
   - For each message, call `get_message_content` then `parse_gemini_transcript`
   - Filter out None results
   - Log count: "Found {N} Gemini transcripts for {date}"
   - Return list of transcript dicts
