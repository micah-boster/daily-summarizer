  <files>src/ingest/transcripts.py</files>
  <action>
Add Gong-specific functions to the existing `src/ingest/transcripts.py` file (Plan 01 creates the file with Gemini functions; this task appends Gong functions):

1. `parse_gong_transcript(message: dict, config: dict) -> dict | None` — Parse a Gong notification email into a transcript dict:
   - Extract headers using `extract_headers` from gmail.py
   - Extract body text using `extract_body_text` from gmail.py
   - If body is empty, return None
   - Parse call/meeting title from subject line. Gong subjects typically contain "Call with [Name]" or "Conversation: [Topic]" — strip common Gong prefixes. Use regex to extract the core meeting identifier.
   - Parse meeting datetime from email Date header using `dateutil.parser.parse`
   - If config `transcripts.preprocessing.strip_filler` is true, apply `strip_filler` to body
   - Return dict: `{"source": "gong", "title": str, "meeting_time": datetime, "transcript_text": str, "raw_email": dict, "message_id": str}`

2. `fetch_gong_transcripts(service, target_date: date, config: dict) -> list[dict]` — Orchestrator function:
   - Read sender_patterns and subject_patterns from `config["transcripts"]["gong"]`
   - Build query using `build_transcript_query`
   - Search messages using `search_messages`
   - For each message, call `get_message_content` then `parse_gong_transcript`
   - Filter out None results
   - Log count: "Found {N} Gong transcripts for {date}"
   - Return list of transcript dicts

3. `fetch_all_transcripts(service, target_date: date, config: dict) -> list[dict]` — Combined function:
   - Call `fetch_gemini_transcripts` and `fetch_gong_transcripts`
   - Combine results into single list
   - Log total: "Total transcripts for {date}: {N} ({gemini_count} Gemini, {gong_count} Gong)"
   - Return combined list (normalization pipeline in Plan 03 handles deduplication and source priority)

IMPORTANT: This task APPENDS to transcripts.py, it does NOT overwrite the Gemini functions from Plan 01. Use the existing file as starting point. If Plan 01 has not yet run (file doesn't exist), create the file with both Gemini and Gong functions — but the PLAN 01 functions are the canonical source.
