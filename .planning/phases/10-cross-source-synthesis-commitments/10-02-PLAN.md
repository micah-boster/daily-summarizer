This module provides `extract_commitments()` which makes a second Claude API call using `output_config` structured outputs to extract commitments from the already-synthesized daily text.

```python
"""Structured commitment extraction via Claude structured outputs.

Second-stage extraction: takes the already-synthesized daily summary text
and extracts commitments as guaranteed-valid JSON using constrained decoding.
"""
from __future__ import annotations

import json
import logging
from datetime import date

import anthropic
from pydantic import BaseModel, ConfigDict, Field

logger = logging.getLogger(__name__)
```

Define models (use ConfigDict(extra="forbid") for structured output compatibility):

1. `ExtractedCommitment(BaseModel)`:
   - `who: str` — Person name, or "TBD" if unclear
   - `what: str` — Commitment description
   - `by_when: str` — ISO date, "week of YYYY-MM-DD", or "unspecified"
   - `source: list[str]` — e.g. ["standup", "Slack #proj-alpha"]
   - `model_config = ConfigDict(extra="forbid")`

2. `CommitmentsOutput(BaseModel)`:
   - `commitments: list[ExtractedCommitment] = Field(default_factory=list)`
   - `model_config = ConfigDict(extra="forbid")`

Define COMMITMENT_EXTRACTION_PROMPT (see 10-RESEARCH.md Pattern 2 for the full prompt template). Key elements:
- Takes `{target_date}` and `{synthesis_text}` format placeholders
- Defines what IS a commitment (explicit "I will" / "X will" / "X agreed to")
- Defines what is NOT (suggestions, observations)
- Date normalization rules relative to target_date
- Rules: extract everyone's commitments, dedup across sources, include partial with gaps

Implement `extract_commitments(synthesis_text: str, target_date: date, config: dict) -> list[ExtractedCommitment]`:
- Build prompt from template with target_date and synthesis_text
- Get model from config.get("synthesis", {}).get("model", "claude-sonnet-4-20250514")
- Call `client.messages.create()` with `output_config={"format": {"type": "json_schema", "schema": CommitmentsOutput.model_json_schema()}}`
- Parse response: `json.loads(response.content[0].text)` -> `CommitmentsOutput.model_validate(data)`
- Return result.commitments
- Wrap in try/except: on any error, log warning and return empty list (graceful degradation — commitment extraction failure should NOT block pipeline)

IMPORTANT: If the anthropic SDK version doesn't support `output_config`, fall back to `extra_headers={"anthropic-beta": "output-format-2025-01-24"}` with `output_format={"type": "json_schema", "schema": ...}`. Test which approach works and use it. The key is guaranteed JSON output.

extracted_commitments = []
try:
    from src.synthesis.commitments import extract_commitments
    # Build the full synthesis text from the synthesis_result sections
    synthesis_text_parts = []
    if synthesis_result.get("executive_summary"):
        synthesis_text_parts.append(synthesis_result["executive_summary"])
    for section in ["substance", "decisions", "commitments"]:
        for item in synthesis_result.get(section, []):
            synthesis_text_parts.append(item)
    synthesis_text = "\n".join(synthesis_text_parts)
    if synthesis_text.strip():
        extracted_commitments = extract_commitments(synthesis_text, target_date, config)
