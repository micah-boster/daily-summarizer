1. Create `config/config.yaml` with default configuration:
   ```yaml
   pipeline:
     timezone: "America/New_York"
     output_dir: "output"
   calendars:
     ids:
       - "primary"
     exclude_patterns: []
   ```

2. Create `src/config.py`:
   - `load_config(config_path: Path | None = None) -> dict` that loads YAML from given path (default: config/config.yaml), applies env var overrides:
     - SUMMARIZER_TIMEZONE -> pipeline.timezone
     - SUMMARIZER_CALENDAR_IDS (comma-separated) -> calendars.ids
     - SUMMARIZER_OUTPUT_DIR -> pipeline.output_dir
   - Use `yaml.safe_load()` (never `yaml.load()`)
   - Return empty dict if config file doesn't exist (graceful default)

3. Create `src/output/__init__.py` (empty).

4. Create `src/output/writer.py`:
   - `create_jinja_env(template_dir: Path) -> Environment` that sets up Jinja2 with `trim_blocks=True, lstrip_blocks=True, keep_trailing_newline=True` and custom filters:
     - `format_time`: datetime -> "2:00pm" format using `strftime("%-I:%M%p").lower()`
     - `format_attendees`: list[Attendee] -> comma-separated names (excluding self), falling back to email if no name
     - `format_duration`: minutes -> "30 min" or "1h 30min" format
   - `write_daily_summary(synthesis: DailySynthesis, output_dir: Path, template_dir: Path) -> Path` that:
     - Renders `daily.md.j2` template with the DailySynthesis data
     - Creates directory structure: `output_dir/daily/YYYY/MM/`
     - Writes to `YYYY-MM-DD.md` in that directory
     - Overwrites if file exists (latest run wins per user decision)
     - Returns the path to the written file

5. Create `templates/daily.md.j2`:
   The template must produce narrative block format (NOT bullets or tables) per user decision. Neutral log tone, third-person factual.

   Structure:
   - `# Daily Summary: {{ date }}` heading
   - `## Overview` with meeting count, total hours, transcript count
   - `## All-Day Events` (conditional -- only if all_day_events exist): iterate events, show title and event_type if not default
   - `## Calendar` section: iterate timed_events chronologically, each as a narrative paragraph: "**{title}**, {start}-{end} ({duration} min). With {attendees}. [Recurring] {description snippet if present}. {meeting link if present}"
   - `## Declined` (conditional): similar narrative for declined events
   - `## Cancelled` (conditional): similar narrative for cancelled events
   - `## Substance` stub section with "No transcript data available yet." if items empty
   - `## Decisions` stub section with same placeholder
   - `## Commitments` stub section with same placeholder
   - Footer: `---` and `*Generated: {{ generated_at }}*`

   Use `{%- ... -%}` or trim_blocks to control whitespace. Tag recurring events with `[Recurring]` annotation.

6. Create `src/main.py` as CLI entry point:
   - `parse_args() -> argparse.Namespace` with:
     - `--from` (dest="from_date", type=date.fromisoformat, default=today)
     - `--to` (dest="to_date", type=date.fromisoformat, default=None, falls back to from_date)
     - `--config` (type=str, default="config/config.yaml")
   - `def main()` that:
     - Parses args, loads config
     - For each date in range: creates a DailySynthesis with empty event lists (calendar ingestion comes in Plan 02)
     - Calls write_daily_summary for each date
     - Logs what was written
   - `if __name__ == "__main__": main()` block
   - Set up basic logging (INFO level to stdout)

7. Create `tests/test_writer.py`:

   - Test output contains Overview section with meeting count
   - Test template renders timed events with narrative format
   - Test template renders all-day events section when present
   - Test template omits Declined section when no declined events
   - Test stub sections appear with placeholder text
   - Use mock DailySynthesis with 2-3 sample NormalizedEvents to test rendering
