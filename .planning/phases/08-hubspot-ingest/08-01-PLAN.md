class SourceType(StrEnum):
    HUBSPOT_DEAL = "hubspot_deal"
    HUBSPOT_CONTACT = "hubspot_contact"
    HUBSPOT_TICKET = "hubspot_ticket"
    HUBSPOT_ACTIVITY = "hubspot_activity"

class ContentType(StrEnum):
    NOTE = "note"
    STAGE_CHANGE = "stage_change"
    ACTIVITY = "activity"

class SourceItem(BaseModel):
    id: str
    source_type: SourceType
    content_type: ContentType
    title: str
    timestamp: datetime
    content: str
    summary: str | None = None
    participants: list[str] = Field(default_factory=list)
    source_url: str
    context: dict = Field(default_factory=dict)
    display_context: str = ""
    raw_data: dict | None = None

    def attribution_text(self) -> str:
        if self.display_context:
            return f"(per {self.display_context})"
        return f"(per {self.source_type.value})"

**Implementation file (`src/ingest/hubspot.py`):**

Module structure:

```python
"""HubSpot CRM ingestion module.

Fetches deal activity, contact notes, tickets, and engagements
(calls, emails, meetings, notes, tasks) for a target date.
Converts to SourceItem objects for synthesis.
"""

# Key functions:
def build_hubspot_client(token=None) -> HubSpot
def _date_to_ms_range(target_date, tz_name) -> tuple[int, int]
def _build_stage_map(client) -> dict[str, str]
def _build_owner_map(client) -> dict[str, str]
def _resolve_owner_id(client, config) -> str | None  # Returns owner_id for "mine" scope
def _search_objects(search_fn, filters, properties, limit) -> list  # Paginated helper
def _fetch_deals(client, start_ms, end_ms, config, stage_map, owner_map) -> list[SourceItem]
def _fetch_contacts(client, start_ms, end_ms, config, owner_map) -> list[SourceItem]
def _fetch_tickets(client, start_ms, end_ms, config, owner_map) -> list[SourceItem]
def _fetch_engagements(client, start_ms, end_ms, config, owner_map) -> list[SourceItem]
def fetch_hubspot_items(config, target_date=None) -> list[SourceItem]  # Main entry point
```

Implementation details:

- `build_hubspot_client`: Read HUBSPOT_ACCESS_TOKEN from env, create HubSpot(access_token=token)
- `_date_to_ms_range`: Convert target date + timezone to start/end millisecond timestamps using zoneinfo
- `_build_stage_map`: Call client.crm.pipelines.pipelines_api.get_all("deals"), build {stage_id: label} dict
- `_build_owner_map`: Call client.crm.owners.owners_api.get_page(limit=500), build {owner_id: name} dict
- `_resolve_owner_id`: If scope is "mine", get authenticated user's owner ID via owners API
- `_search_objects`: Generic paginated search wrapper that handles `after` cursor
- `_fetch_deals`: Search deals by hs_lastmodifieddate BETWEEN start/end. For each deal, fetch with propertiesWithHistory=["dealstage"]. Detect stage changes on target date. Build SourceItem with:
  - source_type=HUBSPOT_DEAL, content_type=STAGE_CHANGE (if stage changed) or ACTIVITY
  - title=deal name, content=formatted deal details (amount, stage, owner, changes)
  - display_context="HubSpot deal {dealname}"
  - source_url=portal_url + "/deal/{id}" or ""
- `_fetch_contacts`: Search contacts by hs_lastmodifieddate. Include associations=["companies"]. For each contact, fetch associated engagements (notes, calls, emails, meetings). Build SourceItem with:
  - source_type=HUBSPOT_CONTACT
  - title="Activity on {firstname} {lastname}"
  - display_context="HubSpot contact {name} ({company})"
- `_fetch_tickets`: Search tickets by hs_lastmodifieddate. Build SourceItem with:
  - source_type=HUBSPOT_TICKET
  - content_type=STAGE_CHANGE (if status changed) or ACTIVITY
  - display_context="HubSpot ticket {subject}"
- `_fetch_engagements`: For each type (notes, calls, emails, meetings, tasks), search by hs_lastmodifieddate. Fetch associations to determine attribution object. Apply deal > contact > company hierarchy. Build SourceItem with:
  - source_type=HUBSPOT_ACTIVITY
  - content_type varies (NOTE for notes, ACTIVITY for others)
  - Calls and meetings get full content; emails and tasks get brief content per user decision
  - display_context="HubSpot {type} on {attributed object}"
- `fetch_hubspot_items`: Main entry point. If not enabled, return []. Build client, lookup maps, ms range. Call all sub-fetchers. Apply volume caps. Return combined list.

Key implementation notes:
- Ownership filter: if config scope is "mine", add hubspot_owner_id filter. If list of IDs, use IN operator.
- Stage name resolution: Use _build_stage_map to convert stage IDs to human-readable names.
- Owner name resolution: Use _build_owner_map to convert owner IDs to names.
- Attribution hierarchy: When an engagement is associated with both a deal and a contact, attribute to the deal only.
- Volume caps: After fetching, truncate each list to config max. Log "... and N more" if truncated.
- Error handling: Wrap each sub-fetcher in try/except, log warning on failure, continue with other types.
