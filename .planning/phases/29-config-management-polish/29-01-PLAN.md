  <name>Task 1: Config API endpoints with atomic writes</name>
  <files>src/api/routers/config.py, src/api/app.py</files>
  <action>
Create `src/api/routers/config.py` with two endpoints:

**GET /config**: Load config via `load_config()`, call `config.model_dump()`, redact sensitive fields (`hubspot.access_token` and `notion.token` replaced with `"••••••"` if non-empty, empty string left as-is). Return the redacted dict.

**PUT /config**: Accept a JSON body (dict). Check if a pipeline run is active by querying `pipeline_runs` table for `status = 'running'` -- if active, return 409 with `{"detail": "Config locked while pipeline is running"}`. Then validate through `PipelineConfig(**body)` -- on `ValidationError`, catch it, call `_format_validation_error(exc)` for the message, and also build a structured `errors` list with `{"field": "dotted.path", "message": "error msg"}` for each error in `exc.errors()`. Return 422 with `{"detail": formatted_msg, "errors": errors_list}`. For sensitive fields: if the PUT body has `"••••••"` for `hubspot.access_token` or `notion.token`, merge the existing value from the current config file before validation. On success, do atomic write: `shutil.copy2(config_path, config_path.with_suffix('.yaml.bak'))` for backup, then `tempfile.mkstemp(dir=config_path.parent, suffix='.yaml.tmp')`, write with `yaml.safe_dump(data, default_flow_style=False)`, then `os.rename(tmp_path, config_path)`. Return 200 with the new config (redacted).

Register the router in `src/api/app.py`: import `config_router` and `application.include_router(config_router, prefix="/api/v1")`.

Use the connection-per-call pattern from pipeline_runner for the pipeline-running check (import sqlite3, open connection to entity db_path, query, close).

**ui-store.ts**: Add `configPanelOpen: boolean` (default false), `toggleConfigPanel: () => void`, `setConfigPanelOpen: (open: boolean) => void` to UIState. Do NOT persist this field (it is ephemeral UI state but the store already persists -- add it to the state, it will persist with session which is fine).

**use-config.ts**: Create TanStack Query hooks following the existing pattern in use-entities.ts / use-summaries.ts:
- `useConfig()` -- `useQuery({ queryKey: ["config"], queryFn: () => apiFetch<Record<string, any>>("/config") })`
- `useUpdateConfig()` -- `useMutation` calling `apiMutate("/config", { method: "PUT", body: data })`, on success invalidate `["config"]` query and show success toast via sonner. On error, parse the error response to extract `errors` array for inline field display.

**config-panel.tsx**: Build a Sheet-based slide-over panel (right side, 540px wide) using the shadcn Sheet component (same pattern as entity-form-panel.tsx). Group config into 5 sections per research recommendations:

1. **Pipeline Settings** -- timezone (text input), output_dir (text input)
2. **Sources** -- Each source (Slack, Google Docs, HubSpot, Notion) gets an enable/disable Switch as header, with detail fields shown only when enabled. Calendars always shown (no toggle). Fields: text inputs for strings, number inputs for ints/floats, Switch for bools, textarea for list fields (one item per line, split on newlines for submission).
3. **Transcripts** -- gemini_drive enabled toggle, pattern lists as textareas, matching settings as number inputs
4. **Synthesis** -- model (text input), token limits (number inputs), concurrency (number input)
5. **Processing** -- dedup (enabled + threshold + log_dir), entity (enabled + db_path + thresholds), cache (ttl days)

Each section is a collapsible card (ChevronDown/Up icon). Use the existing `collapsedSections` pattern from ui-store for section collapse state.

Inline validation: maintain a `fieldErrors: Record<string, string>` state. On mutation error, parse the `errors` array from the 422 response and set field errors. Show red text below each errored field. Clear field errors on value change.

Pipeline lock banner: Use `usePipelineStore` to check `status === "running"`. When running, show an amber/yellow banner at top of panel "Config locked while pipeline is running" and disable all inputs via a wrapping fieldset with `disabled` attribute.

Save button at bottom: disabled when form is pristine (no changes) or pipeline is running. Uses the `useUpdateConfig` mutation. Show loading spinner during save.

**status-bar.tsx**: Add a Settings (gear) icon button to the right side of the status bar (before the elapsed time display). Import `Settings` from lucide-react. On click, call `useUIStore.getState().toggleConfigPanel()`. Use `Button variant="ghost" size="icon"` with `h-7 w-7` sizing to match the status bar scale.
