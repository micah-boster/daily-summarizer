  <action>
**ui-store.ts**: Add keyboard navigation state to UIState:
- `focusedColumn: "left" | "center" | "right"` (default `"center"`)
- `focusedIndex: number` (default `-1`, meaning no item focused)
- `shortcutHelpOpen: boolean` (default `false`)
- `setFocusedColumn: (col: "left" | "center" | "right") => void`
- `setFocusedIndex: (idx: number) => void`
- `toggleShortcutHelp: () => void`

Do NOT persist these (ephemeral). They are already inside the persisted store, which uses sessionStorage -- this is acceptable since they reset each session anyway.

**use-keyboard-nav.ts**: Create a custom hook `useKeyboardNav()` that:

1. Adds a single `document.addEventListener("keydown", handler)` in a `useEffect` with proper cleanup.

2. The handler first checks if the active element is an input, textarea, select, or has `contenteditable`. If so, return early (do not process). Also skip if a dialog/modal is open (check for `[role="dialog"]` in DOM or if `commandPaletteOpen` is true in ui-store).

3. Key mappings:
   - `j`: Increment `focusedIndex` (clamp to list length). The list length depends on `focusedColumn` -- this hook does not know list lengths directly. Instead, use `data-kb-item` attributes on navigable items and `document.querySelectorAll('[data-kb-column="left"] [data-kb-item]')` to determine list length for the active column. Scroll the focused item into view.
   - `k`: Decrement `focusedIndex` (clamp to 0).
   - `h`: Move `focusedColumn` left (right -> center -> left). Reset `focusedIndex` to -1.
   - `l`: Move `focusedColumn` right (left -> center -> right). Reset `focusedIndex` to -1.
   - `Enter`: If `focusedIndex >= 0`, click the focused item (find `[data-kb-column="X"] [data-kb-item]` at index and call `.click()`).
   - `Escape`: If `focusedIndex >= 0`, reset to -1. If already -1, reset column to "center".
   - `?`: Toggle `shortcutHelpOpen` in ui-store.
   - `r`: Trigger pipeline run -- call `apiMutate("/runs", { method: "POST" })` and show toast. Check pipeline store status first; if already running, show "Pipeline already running" toast instead.
   - Do NOT intercept `Meta+k` / `Ctrl+k` -- let the existing command palette handler handle it.

4. Apply `data-kb-focused` attribute to the currently focused item via a second `useEffect` that reacts to `focusedColumn` and `focusedIndex` changes. Use `ring-2 ring-primary/50` class on the focused element. Remove it from the previous element.

**app-shell.tsx**:
- Call `useKeyboardNav()` at the top of the component.
- Add `data-kb-column="left"`, `data-kb-column="center"`, `data-kb-column="right"` attributes to the three column wrapper divs.
- Add a subtle visual indicator for the focused column -- e.g., a thin `border-primary/20` on the focused column div (read `focusedColumn` from ui-store).
