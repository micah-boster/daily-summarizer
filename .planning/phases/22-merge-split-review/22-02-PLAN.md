In src/entity/merger.py, implement:

execute_merge(repo: EntityRepository, source_entity_id: str, target_entity_id: str, score: float = 0.0) -> None:
  1. Verify both entities exist and are active
  2. repo.reassign_mentions(source_entity_id, target_entity_id)
  3. repo.transfer_aliases(source_entity_id, target_entity_id)
  4. Get source entity name, add as alias on target: try repo.add_alias(target_entity_id, source_name) except ValueError (collision): pass
  5. Soft-delete source and set merge_target_id:
     repo._conn.execute("UPDATE entities SET deleted_at=?, merge_target_id=?, updated_at=? WHERE id=?", (now, target_entity_id, now, source_entity_id))
     repo._conn.commit()
  6. Save approved proposal: repo.save_proposal(source_entity_id, target_entity_id, reason=f"merged (score {score:.0f})", status="approved")

execute_split(repo: EntityRepository, entity_id: str) -> None:
  1. Find merge history: proposals = repo.get_merge_history(entity_id)
  2. If no approved merges: raise ValueError("Entity was never merged")
  3. Get the most recent merge proposal: last_merge = proposals[0]
  4. target_entity_id = last_merge.target_entity_id (the entity that absorbed this one)
  5. Verify entity is soft-deleted: entity = repo._conn.execute("SELECT * FROM entities WHERE id = ? AND deleted_at IS NOT NULL", ...). If not found: raise ValueError("Entity is not merged/deleted")
  6. Restore entity: repo.restore_entity(entity_id)
  7. Re-attribute mentions:
     - Get source entity's name and all aliases (after restoration)
     - For each mention currently on target_entity_id:
       - Check if the mention's source_type + original context matches the restored entity's names
       - Simple approach: use the entity's canonical name + aliases to identify which mentions belong to it
       - For mentions that can't be unambiguously attributed: leave on target (safe default)
     - Implementation: query mentions on target, for each check if context_snippet or source name matches restored entity names via normalize_for_matching
  8. Move back aliases:
     - The source entity's canonical name was added as alias on target during merge -- remove it from target
     - Aliases that were on source before merge: identify by checking alias values against source entity name patterns
     - Practical approach: the merge_proposal records source_entity_id. Get aliases on target that match source entity name patterns. Move those back.
     - Simplest reliable approach: remove source's canonical name alias from target; transfer aliases that were originally on source (track by checking alias creation timestamp vs merge timestamp)
  9. Log the split action
