  <action>
Create `src/synthesis/validator.py` with:

1. `BANNED_PATTERNS` -- a list of compiled regex patterns that catch evaluative language. Categories:

**Performance adjectives applied to people/work:**
- `\b(?:productive|effective|impressive|excellent|poor|weak|strong|brilliant|incompetent)\b`
- `\b(?:good|great|bad|terrible|wonderful|awful|amazing|outstanding)\s+(?:job|work|effort|performance|leadership|contribution|insight)\b`

**Editorial/speculative language:**
- `\b(?:clearly|obviously|unfortunately|fortunately|surprisingly|remarkably|notably)\b`
- `\b(?:seemed|appeared|felt)\s+(?:to be|like|as if)\b`
- `\b(?:it(?:'s| is) (?:clear|obvious|evident|apparent) that)\b`

**Evaluative framing:**
- `\b(?:wisely|foolishly|rightly|wrongly|correctly|incorrectly)\s+(?:decided|chose|opted)\b`
- `\b(?:strong|weak|excellent|poor)\s+(?:decision|choice|move|strategy|approach)\b`
- `\b(?:showed|demonstrated|displayed|exhibited)\s+(?:leadership|initiative|competence|skill)\b`

**Implications about individuals:**
- `\b(?:struggled|excelled|thrived|faltered|dominated|failed to)\b` (when applied to people, not systems)
- `\b(?:should have|could have|ought to have)\b` (hindsight judgment)

2. `class ValidationViolation(BaseModel)` with fields: `text` (the matched text), `pattern` (which pattern matched), `context` (surrounding 50 chars for debugging).

3. `def validate_evidence_only(text: str) -> list[ValidationViolation]`: Scan text against all BANNED_PATTERNS. For each match, extract the matched text, the pattern name, and surrounding context. Return list of violations. Empty list means text passes validation.

4. `def validate_source_attribution(text: str) -> list[str]`: Check that synthesis items have inline parenthetical attribution. Scan for bullet points or items that lack a `(Meeting Title` pattern. Return list of items missing attribution. This is only used on synthesis output (not extraction).

5. `def is_clean(text: str) -> bool`: Convenience function. Returns True if validate_evidence_only returns empty list.

Design the patterns to minimize false positives. The word "strong" alone is fine ("strong coffee"), but "strong leadership" is evaluative. The patterns above are scoped to avoid common false positives.
