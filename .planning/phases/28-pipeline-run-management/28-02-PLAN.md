  <action>
    Create web/src/stores/pipeline-store.ts as a Zustand store (NO persist -- pipeline state is ephemeral):
    ```typescript
    interface PipelineStage {
      name: string;
      status: "pending" | "running" | "complete" | "failed";
      elapsed_s: number | null;
    }

    interface PipelineState {
      runId: string | null;
      status: "idle" | "running" | "complete" | "failed";
      targetDate: string | null;
      stages: PipelineStage[];
      elapsedS: number;
      error: string | null;
      errorDismissed: boolean;

      startRun: (runId: string, targetDate: string) => void;
      updateFromEvent: (event: PipelineEvent) => void;
      reset: () => void;
      dismissError: () => void;
    }
    ```
    - startRun sets runId, targetDate, status="running", clears stages/error
    - updateFromEvent updates stages, status, elapsed_s, error from SSE event
    - reset returns to idle state
    - dismissError sets errorDismissed=true (hides red error indicator)

    Create web/src/hooks/use-pipeline.ts with usePipelineRun hook:
    - triggerRun(targetDate?: string): POST /api/v1/runs with target_date. On 202, call store.startRun(). Open EventSource to /api/v1/runs/{runId}/stream. On each message, call store.updateFromEvent(). On "complete" status, close EventSource.
    - EventSource error handler: EventSource auto-reconnects. On permanent failure (readyState === CLOSED), mark as connection error.
    - On "complete" event: show toast("Pipeline complete for {date}", { action: { label: "View Summary", onClick: () => navigate to date } }) using sonner. Call TanStack Query invalidation for summary list.
    - On "failed" event: show error toast with error message.
    - On 409 from POST: show toast("A pipeline run is already in progress").
    - Cleanup: close EventSource on unmount.
    - Return { triggerRun, isRunning: status === "running" }

    Check if web/src/lib/api.ts exists. If so, add API_BASE constant there if not already present. If not, create it with:
    ```typescript
    export const API_BASE = "http://localhost:8000/api/v1";
    ```
    Use this constant in the hook for fetch and EventSource URLs.

    For the success toast auto-navigation: use the existing setSelectedDate and setSelectedViewType from useUIStore to navigate to the completed date's summary.
