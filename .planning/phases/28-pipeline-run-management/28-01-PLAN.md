  <action>
    Create src/api/routers/pipeline.py with these endpoints:

    POST /runs -- Trigger a new pipeline run
    - Accept TriggerRequest body (target_date defaults to yesterday)
    - Call create_run() to get run_id (returns 409 Conflict if already running)
    - Call start_pipeline_subprocess() in background (asyncio.create_task wrapping the event reader)
    - Store active process handle in module-level dict keyed by run_id
    - Return TriggerResponse with run_id and status="running" (202 Accepted)

    GET /runs -- List run history
    - Accept optional limit query param (default 14)
    - Call list_runs() and return RunListResponse

    GET /runs/{run_id} -- Get single run details
    - Call get_run(), return RunResponse or 404

    GET /runs/{run_id}/stream -- SSE stream for run progress
    - Return StreamingResponse with media_type="text/event-stream"
    - Headers: Cache-Control=no-cache, Connection=keep-alive, X-Accel-Buffering=no
    - First event: current run state from get_run() (full state snapshot for reconnection)
    - Then: if process is still active, read events from the active process stdout
    - If process already complete/failed, send final state and close
    - Each event formatted as: f"data: {json.dumps(event)}\n\n"
    - On client disconnect: stop reading (check if client is still connected via asyncio)

    This endpoint MUST use `async def` (not sync def) since it uses StreamingResponse with async generator. Other endpoints can be sync def following project convention.

    Register router in src/api/app.py:
