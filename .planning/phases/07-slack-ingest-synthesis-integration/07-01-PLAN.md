class SourceType(StrEnum):
    SLACK_MESSAGE = "slack_message"
    SLACK_THREAD = "slack_thread"

class ContentType(StrEnum):
    MESSAGE = "message"
    THREAD = "thread"

class SourceItem(BaseModel):
    id: str
    source_type: SourceType
    content_type: ContentType
    title: str
    timestamp: datetime
    content: str
    summary: str | None = None
    participants: list[str] = Field(default_factory=list)
    source_url: str
    context: dict = Field(default_factory=dict)
    display_context: str = ""
    raw_data: dict | None = None

    def attribution_text(self) -> str:
        if self.display_context:
            return f"(per {self.display_context})"
        return f"(per {self.source_type.value})"

**Create `src/ingest/slack.py`** with these functions:

1. `build_slack_client(token: str | None = None) -> WebClient` ŌĆö Initialize WebClient with bot token from param or `SLACK_BOT_TOKEN` env var. Attach `RateLimitErrorRetryHandler(max_retry_count=2)`. Raise ValueError if no token found.

2. `resolve_user_names(client: WebClient, user_ids: set[str]) -> dict[str, str]` ŌĆö Batch resolve user IDs to display names. Call `users_info` for each unknown ID. Cache results in a module-level dict `_user_cache: dict[str, str] = {}` to avoid repeated lookups across channels. Use `profile.display_name` if set, else `real_name`, else user_id as fallback.

3. `load_slack_state(config_dir: Path) -> dict` ŌĆö Load `config/slack_state.json` if exists, else return empty structure `{"channels": {}, "dms": {}}`. Timestamps are strings.

4. `save_slack_state(state: dict, config_dir: Path) -> None` ŌĆö Write slack_state.json atomically.

5. `fetch_channel_messages(client: WebClient, channel_id: str, oldest: str, latest: str) -> list[dict]` ŌĆö Cursor-paginated `conversations_history` with limit=200. Return all messages in the time window.

6. `fetch_thread_replies(client: WebClient, channel_id: str, thread_ts: str) -> list[dict]` ŌĆö Cursor-paginated `conversations_replies` with limit=200. **Skip the first message on the first page** (it's the parent, already in channel history). Return only reply messages.

7. `should_expand_thread(msg: dict, config: dict) -> bool` ŌĆö Check `reply_count >= config.slack.thread_min_replies` (default 3) AND `reply_users_count >= config.slack.thread_min_participants` (default 2). Both conditions must be met (per user decision).

8. `message_to_source_item(msg: dict, channel_name: str, channel_id: str, user_map: dict[str, str], is_dm: bool = False, dm_partner: str | None = None) -> SourceItem` ŌĆö Convert a raw Slack message dict to a SourceItem:
   - `id`: `f"slack_{channel_id}_{msg['ts']}"`
   - `source_type`: `SourceType.SLACK_MESSAGE`
   - `content_type`: `ContentType.MESSAGE`
   - `title`: `f"Message from {user_name}"`
   - `timestamp`: `datetime.fromtimestamp(float(msg['ts']), tz=timezone.utc)`
   - `content`: `msg.get('text', '')`
   - `participants`: `[user_name]`
   - `source_url`: `f"https://slack.com/archives/{channel_id}/p{msg['ts'].replace('.', '')}"`
   - `display_context`: `f"Slack DM with {dm_partner}"` if is_dm else `f"Slack #{channel_name}"`
   - `context`: `{"channel_id": channel_id, "thread_ts": msg.get("thread_ts")}`
   - `raw_data`: the full msg dict

9. `thread_to_source_item(parent_msg: dict, replies: list[dict], channel_name: str, channel_id: str, user_map: dict[str, str]) -> SourceItem` ŌĆö Convert an expanded thread to a single SourceItem:
   - `source_type`: `SourceType.SLACK_THREAD`
   - `content_type`: `ContentType.THREAD`
   - `title`: `f"Thread: {parent_text[:80]}..."` (truncate parent text)
   - `content`: Format as parent text + reply texts joined with newlines, each prefixed with sender name
   - `participants`: deduplicated list of all participants in thread
   - Reply count hint in context: `{"reply_count": len(replies), "channel_id": channel_id}`

10. `fetch_slack_items(config: dict) -> list[SourceItem]` ŌĆö Main orchestrator:
    - Build client from SLACK_BOT_TOKEN env var
    - Load slack_state.json for last-ingested timestamps
    - For each configured channel ID in `config.slack.channels`:
      - Determine oldest timestamp (from state or 24h lookback for first run)
      - Determine latest timestamp (now)
      - Fetch messages via `fetch_channel_messages`
      - Resolve channel name via `conversations_info` (cache it)
      - Resolve user names for all unique user IDs in messages
      - Filter messages through `should_keep_message` from slack_filter.py
      - For messages with threads meeting expansion threshold: fetch replies, create thread SourceItem
      - For remaining messages: create individual SourceItems
      - Apply volume cap: if messages exceed `config.slack.max_messages_per_channel` (default 100), keep only the most recent N
      - Update state with latest message timestamp
    - For each configured DM ID in `config.slack.dms`:
      - Same flow as channels but with `is_dm=True` and resolve DM partner name
      - For group DMs (mpim): format partner as "Name1, Name2, ..." or "group DM" if 4+
    - Save updated state
    - Return all SourceItems sorted by timestamp
