  <files>src/ingest/slack_discovery.py, tests/test_slack_discovery.py</files>
  <action>
Create `src/ingest/slack_discovery.py` with these functions:

1. `get_user_channels(client: WebClient) -> list[dict]` — Call `users_conversations(types="public_channel,private_channel", limit=200)` with cursor pagination. Return list of channel dicts with keys: id, name, is_private, num_members.

2. `get_user_dms(client: WebClient) -> list[dict]` — Call `users_conversations(types="im,mpim", limit=200)` with cursor pagination. Return list of DM channel dicts. For each, resolve the partner name(s) using `resolve_user_names` from slack.py.

3. `compute_channel_stats(client: WebClient, channel_id: str, lookback_days: int = 7) -> dict` — Fetch recent messages (last N days) from the channel. Compute:
   - `message_count`: total messages
   - `participant_count`: unique user IDs
   - `topic_keywords`: Extract 2-3 most common non-stopword, non-trivial words from message text (simple word frequency, no NLP needed). Use a small stopword set: {"the", "a", "an", "is", "are", "was", "were", "it", "to", "in", "for", "on", "with", "at", "by", "from", "of", "and", "or", "but", "not", "this", "that", "i", "we", "you", "they", "he", "she", "have", "has", "had", "do", "does", "did", "will", "would", "can", "could", "should", "be", "been", "being"}.
   Only propose channels with message_count > 5 in the lookback window (pre-filter low-activity channels per research guidance).

4. `discover_channels(client: WebClient, state: dict, config: dict) -> list[str]` — Main channel discovery flow:
   - Get all user channels
   - Separate into already_configured (in state or config) and unconfigured
   - Show already-configured channels first: "Currently tracking: #channel-name (47 messages last 7d)"
   - For unconfigured channels with recent activity (message_count > 5):
     - Compute stats
     - Present one at a time: "Add #channel-name? (47 messages, 8 participants, topics: redesign, sprint, figma) [y/n/q]"
     - y = add, n = skip, q = quit discovery
   - Return list of all confirmed channel IDs (existing + newly added)

5. `discover_dms(client: WebClient, state: dict, config: dict) -> list[str]` — DM discovery flow:
   - Get all user DMs
   - Filter to DMs with recent activity (same message_count > 5 threshold)
   - Present each active DM: "Add DM with Sarah Chen? (23 messages last 7d) [y/n/q]"
   - For group DMs: "Add group DM with Sarah, Mike, Alex? (15 messages last 7d) [y/n/q]"
   - Return list of confirmed DM channel IDs

6. `check_new_channels(client: WebClient, state: dict, config: dict) -> list[str]` — Periodic auto-suggest check:
   - Compare user's current workspace channels (via `get_user_channels`) against configured channels in config.slack.channels
   - For unconfigured channels, compute stats (reuse `compute_channel_stats`) and filter to those with message_count > 5
   - Return list of channel names that are active but not tracked
   - This is a non-interactive check — it returns names only, does not prompt
   - Called from the pipeline (wired in Plan 03) when `state.get("last_discovery_check")` is older than `config.slack.discovery_check_days` (default 7) or missing
   - After check, update `state["last_discovery_check"]` to current ISO timestamp

7. `run_discovery(config: dict) -> None` — Full discovery orchestrator:
   - Build Slack client
   - Load current state
   - Run channel discovery
   - Run DM discovery
   - Update config/slack_state.json with new channel/DM list and metadata (name, partner names)
   - Update config.yaml slack.channels and slack.dms lists with confirmed IDs
   - Print summary: "Discovery complete. Tracking N channels and M DMs."
   - Save state

**Interactive input:** Use `input()` for user prompts. Keep prompts concise and scannable. Format:
```
=== Slack Channel Discovery ===

Currently tracking:
  #design-team (47 messages, 8 participants last 7d)
  #eng-standup (32 messages, 5 participants last 7d)

New suggestions:
  Add #product-launches? (23 messages, 6 participants, topics: launch, beta, timeline) [y/n/q]:
```

**Config update:** After discovery, write updated channel/DM IDs back to config.yaml. Use PyYAML to load, update the `slack.channels` and `slack.dms` lists, and write back. Preserve existing config structure.
