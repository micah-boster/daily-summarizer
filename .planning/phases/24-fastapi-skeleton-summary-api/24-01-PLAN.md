    **Step 1: Install FastAPI and uvicorn.**
    Run `uv add fastapi uvicorn`. This adds them to pyproject.toml and installs.

    **Step 2: Add busy_timeout to get_connection().**
    In `src/entity/db.py`, add `conn.execute("PRAGMA busy_timeout=5000")` on the line immediately after `conn.execute("PRAGMA foreign_keys=ON")` (after line 47). This is a one-line change. Do NOT modify anything else in this file.

    **Step 3: Create response models at `src/api/models/responses.py`.**
    Define three Pydantic BaseModel classes:
    - `SummaryListItem`: date (date), meeting_count (int|None=None), commitment_count (int|None=None), has_sidecar (bool=False)
    - `SummaryResponse`: date (date), markdown (str), sidecar (dict|None=None) -- use dict not DailySidecar to avoid coupling response shape to internal model
    - `StatusResponse`: status (str="ok"), db_connected (bool), summary_count (int), last_summary_date (date|None=None)

    **Step 4: Create SummaryReader service at `src/api/services/summary_reader.py`.**
    Class with `__init__(self, output_dir: str = "output")` storing `self._base = Path(output_dir) / "daily"`.
    Methods:
    - `list_available_dates() -> list[date]`: Glob `self._base` for `**/*.md` files, parse date from filename stem via `date.fromisoformat()`, skip non-date filenames, return sorted descending.
    - `list_available_dates_with_previews() -> list[SummaryListItem]`: For each date from list_available_dates(), check if corresponding .json exists, if yes load it and extract meeting_count + commitment_count (use `DailySidecar.model_validate()` to handle older schemas with missing fields). Return list of SummaryListItem.
    - `get_summary(target_date: date) -> SummaryResponse | None`: Build path from `self._base / str(year) / f"{month:02d}" / f"{target_date.isoformat()}.md"`. If markdown file does not exist, return None. Read markdown text. Check for .json sidecar -- if exists, load as dict via json.loads (raw dict, not validated). Return SummaryResponse with date, markdown, and sidecar (or None).

    Import DailySidecar from `src.sidecar` only in `list_available_dates_with_previews` for model_validate. Import SummaryListItem and SummaryResponse from `src.api.models.responses`.

    **Step 5: Create dependency injection at `src/api/deps.py`.**
    - `get_config() -> PipelineConfig`: Wrap in try/except catching SystemExit (because load_config calls sys.exit on bad config). If SystemExit caught, raise `HTTPException(status_code=500, detail="Invalid pipeline configuration")`. Import load_config from src.config.
    - `get_entity_repo()`: Generator function. Gets config via get_config(), creates EntityRepository(config.entity.db_path), calls repo.connect(), yields repo, calls repo.close() in finally block. Import EntityRepository from src.entity.repository.
    - `get_summary_reader() -> SummaryReader`: Gets config via get_config(), returns SummaryReader(output_dir=config.pipeline.output_dir). Import SummaryReader from src.api.services.summary_reader.

    **Step 6: Create summaries router at `src/api/routers/summaries.py`.**
    Create `router = APIRouter(tags=["summaries"])` with three endpoints:
    - `GET /summaries` (sync def): Injects SummaryReader via Depends(get_summary_reader). Returns reader.list_available_dates_with_previews(). Response model: list[SummaryListItem].
    - `GET /summaries/{target_date}` (sync def): target_date is `date` type path param. Injects SummaryReader. Calls reader.get_summary(target_date). If None, raise HTTPException(404, f"No summary found for {target_date}"). Returns SummaryResponse.
    - `GET /status` (sync def): Injects SummaryReader and PipelineConfig via Depends. Checks DB connectivity by creating EntityRepository, connecting, closing in try/except. Returns StatusResponse with db_connected, summary_count from len(reader.list_available_dates()), last_summary_date from first date or None.

    All three endpoints use sync `def` (NOT `async def`) so FastAPI runs them in its thread pool. This is critical -- async def with sync file/DB I/O blocks the event loop.

    **Step 7: Create app factory at `src/api/app.py`.**
    `create_app() -> FastAPI`: Creates FastAPI instance with title="Daily Summarizer API", version="0.1.0", docs_url="/docs". Adds CORSMiddleware FIRST (before any other middleware) with allow_origins=["http://localhost:3000"], allow_methods=["GET", "POST", "PUT", "DELETE"], allow_headers=["Content-Type"], allow_credentials=False. Includes summaries router with prefix="/api/v1". Returns app.
    Module-level: `app = create_app()` so uvicorn can find it at `src.api.app:app`.

    **Step 8: Create all `__init__.py` files** (empty) for: src/api/, src/api/routers/, src/api/services/, src/api/models/.
