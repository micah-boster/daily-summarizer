1. Create `src/models/rollups.py` with Pydantic models:
   - `ThreadEntry(BaseModel)`: date (date), content (str), category (str — "decision"|"commitment"|"substance")
   - `WeeklyThread(BaseModel)`: title (str), significance (str — "high"|"medium"), entries (list[ThreadEntry]), progression (str — narrative arc like "raised -> explored -> decided"), status (str — "resolved"|"open"|"escalated"), tags (list[str])
   - `WeeklySynthesis(BaseModel)`: week_number (int), year (int), start_date (date), end_date (date), generated_at (datetime), daily_count (int), is_partial (bool), meeting_count (int), total_meeting_hours (float), threads (list[WeeklyThread]), single_day_items (list[ThreadEntry] — significant items that don't form multi-day threads), still_open (list[dict] — commitments/questions carrying forward with status), daily_dates (list[date])

2. Create `src/synthesis/weekly.py` with the weekly pipeline:
   - `read_daily_summaries(output_dir: Path, start: date, end: date) -> list[dict]`: Read rendered daily .md files from `output/daily/YYYY/MM/YYYY-MM-DD.md`. For each file, extract ONLY synthesis sections (everything between "## Substance" and "## Meetings Without Transcripts" or "## Calendar", whichever comes first). Also extract "## Executive Summary" if present. Return list of dicts with keys: date, executive_summary (str|None), substance (list[str]), decisions (list[str]), commitments (list[str]), path (Path).
   - `_get_week_date_range(target_date: date) -> tuple[date, date]`: Given any date, compute the Monday-Friday range for that ISO week using `isocalendar()`. Monday = start, Friday = end. If target_date is Saturday/Sunday, use the preceding week.
   - `_build_thread_detection_prompt(summaries: list[dict]) -> str`: Format all daily synthesis content into a prompt for Claude. The prompt instructs Claude to: (a) identify threads — topics/decisions/commitments that appear or evolve across multiple days, (b) trace each thread's progression chronologically, (c) rank threads by significance not frequency, (d) elevate important single-day items alongside threads, (e) list open commitments and unresolved questions with current status. The prompt MUST say: "Only link items that genuinely refer to the same topic. Single-day items are fine if significant. Do NOT invent connections." Output format should be structured markdown with ## Thread sections and ## Single-Day Items and ## Still Open sections.
   - `_parse_weekly_response(response_text: str, summaries: list[dict]) -> tuple[list[WeeklyThread], list[ThreadEntry], list[dict]]`: Parse Claude's thread detection response into WeeklyThread models, single-day ThreadEntry items, and still-open items. Handle the structured markdown sections.
   - `synthesize_weekly(target_date: date, config: dict, output_dir: Path) -> WeeklySynthesis`: Main entry point. Calls `_get_week_date_range`, `read_daily_summaries`, builds prompt, calls Claude API (same pattern as `synthesize_daily` — use `anthropic.Anthropic()`, `client.messages.create`), parses response, runs `validate_evidence_only` on response, constructs and returns WeeklySynthesis. Use model from `config.get("synthesis", {}).get("model", DEFAULT_MODEL)` and `weekly_max_output_tokens` from config (default 8192).

3. Add weekly prompt template to `src/synthesis/prompts.py`:
   - Add `WEEKLY_THREAD_DETECTION_PROMPT` constant string used by `_build_thread_detection_prompt`. The prompt should enforce: neutral reporter tone, evidence-only language, significance-based ranking (not frequency), thread progression narrative (chronological arc), category tags on each thread item (decision/commitment/substance), and explicit "Still Open" tracking for unresolved items.
