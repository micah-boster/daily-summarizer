1. Add `MonthlySynthesis` model to `src/models/rollups.py`:
   - `ThematicArc(BaseModel)`: title (str), description (str — 2-3 sentence analytical summary), weeks_active (list[int] — which week numbers this theme appeared in), trajectory (str — "growing"|"declining"|"stable"|"emerging"|"resolved"), key_moments (list[str] — specific significant events within the arc)
   - `MonthlyMetrics(BaseModel)`: total_meetings (int), total_hours (float), total_decisions (int), top_attendees (list[str] — most frequently appearing people), weekly_meeting_counts (list[int] — meetings per week for trend)
   - `MonthlySynthesis(BaseModel)`: month (int), year (int), generated_at (datetime), weekly_count (int), thematic_arcs (list[ThematicArc]), strategic_shifts (list[str] — notable priority changes or direction shifts), emerging_risks (list[str] — concerns that grew over the month), metrics (MonthlyMetrics), still_open (list[dict] — items still unresolved at month end)

2. Create `src/synthesis/monthly.py` with the monthly pipeline:
   - `read_weekly_summaries(output_dir: Path, year: int, month: int) -> list[dict]`: Read rendered weekly .md files from `output/weekly/YYYY/YYYY-WXX.md`. Determine which ISO weeks fall within the target month (a week belongs to the month if its Monday is in that month). For each weekly file, extract the full content. Return list of dicts with keys: week_number, year, content (str), path (Path).
   - `_aggregate_monthly_metrics(output_dir: Path, year: int, month: int, config: dict) -> MonthlyMetrics`: Read all daily .md files for the month. Parse the overview line ("N meetings, X.X hours") from each daily. Count decision items across all dailies. Extract attendee names from calendar sections. Compute totals.
   - `_build_monthly_narrative_prompt(weeklies: list[dict], metrics: MonthlyMetrics) -> str`: Format all weekly content and metrics into a prompt for Claude. The prompt instructs Claude to: (a) identify 3-5 thematic arcs that span multiple weeks, (b) describe each arc's trajectory (growing/declining/emerging/resolved), (c) note strategic shifts and priority changes, (d) flag emerging risks, (e) write in analytical third-person briefing tone ("Three themes dominated April:..."). The prompt MUST say: "This is NOT a longer weekly. Do NOT chronologically list weeks. Synthesize themes visible only at monthly scale." and "Use analytical third-person tone throughout."
   - `_parse_monthly_response(response_text: str) -> tuple[list[ThematicArc], list[str], list[str]]`: Parse Claude's response into ThematicArc models, strategic_shifts list, and emerging_risks list.
   - `synthesize_monthly(year: int, month: int, config: dict, output_dir: Path) -> MonthlySynthesis`: Main entry point. Calls `read_weekly_summaries`, `_aggregate_monthly_metrics`, builds prompt, calls Claude API, parses response, runs `validate_evidence_only` on response, constructs and returns MonthlySynthesis. Use model from `config.get("synthesis", {}).get("model", DEFAULT_MODEL)` and `monthly_max_output_tokens` from config (default 8192).

3. Add `MONTHLY_NARRATIVE_PROMPT` constant to `src/synthesis/prompts.py`:
   - Enforce: analytical third-person tone, thematic arcs not chronological recaps, trajectory labels, evidence-only language, metric context (not a dashboard, just context for narrative).
