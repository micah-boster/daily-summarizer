<objective>
Migrate weekly.py and monthly.py from free-text Claude API calls with regex parsing to json_schema structured outputs with Pydantic model validation, completing STRUCT-01 coverage for all Claude API call sites.

Purpose: Eliminates the last two fragile regex parsers in the synthesis pipeline. Both weekly.py and monthly.py currently call `_call_claude_with_retry` (plain text) and parse the response with `_parse_weekly_response` / `_parse_monthly_response` (regex-based). This migration replicates the proven pattern from extractor.py and synthesizer.py — Pydantic output models, `output_config` with `json_schema`, and `model_validate` for parsing.

Output: Updated weekly.py and monthly.py using structured outputs, new Pydantic output models in models.py, updated tests covering converter functions and schema validity.
