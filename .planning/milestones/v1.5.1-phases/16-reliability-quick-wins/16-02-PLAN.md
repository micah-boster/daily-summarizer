  <files>src/cache_cleanup.py, src/pipeline.py</files>
  <action>
  1. Create `src/cache_cleanup.py`:
     ```python
     """Cache retention policy: auto-delete old raw data files.

     Runs at pipeline startup before ingest begins. Never touches
     processed output (daily summaries, quality files).
     """
     ```

     Implement `cleanup_raw_cache(output_dir: Path, raw_ttl_days: int = 14, dedup_log_ttl_days: int = 30) -> tuple[int, int]`:
     - Walk `output_dir / "raw"` recursively.
     - For each file: if `time.time() - file.stat().st_mtime > raw_ttl_days * 86400`, delete it and accumulate count + size.
     - After deleting files, clean up empty directories in `output/raw/` (bottom-up).
     - Walk `output_dir / "dedup_logs"` recursively (if exists).
     - For each file: apply `dedup_log_ttl_days` threshold.
     - Return `(files_deleted, bytes_freed)`.
     - CRITICAL: NEVER touch `output_dir / "daily"`, `output_dir / "quality"`, or `output_dir / "validation"`. Only walk `output_dir / "raw"` and `output_dir / "dedup_logs"`.
     - If `output_dir / "raw"` doesn't exist, return (0, 0) immediately.
     - Log a single summary line: `"Cache cleanup: deleted {N} files, freed {bytes_freed} bytes"` (only if N > 0).

  2. In `src/pipeline.py`:
     - Add import: `from src.cache_cleanup import cleanup_raw_cache`
     - At the very start of `run_pipeline()` (before `_ingest_slack`), add:
       ```python
       # Cache retention: cleanup old raw data
       try:
           deleted, freed = cleanup_raw_cache(
               ctx.output_dir,
               raw_ttl_days=ctx.config.cache.raw_ttl_days,
               dedup_log_ttl_days=ctx.config.cache.dedup_log_ttl_days,
           )
           if deleted:
               logger.info("Cache cleanup: deleted %d files, freed %d bytes", deleted, freed)
       except Exception as e:
           logger.warning("Cache cleanup failed: %s. Continuing.", e)
       ```
