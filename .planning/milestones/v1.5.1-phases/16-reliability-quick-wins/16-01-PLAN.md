  In `src/ingest/slack.py`, implement the following changes:

  2. Add a new function `_load_user_cache(cache_path: Path, ttl_days: int) -> dict[str, str] | None`:
     - If cache_path exists and `time.time() - cache_path.stat().st_mtime < ttl_days * 86400`, load and return the JSON dict.
     - Otherwise return None.

  3. Add a new function `_save_user_cache(user_map: dict[str, str], cache_path: Path) -> None`:
     - Write user_map to cache_path as JSON using write-to-temp-then-rename pattern (same as `save_slack_state()`).

  4. Add a new function `_fetch_all_users_batch(client: WebClient) -> dict[str, str]`:
     - Decorated with `@retry_api_call`.
     - Call `client.users_list(limit=1000)` with cursor pagination.
     - Filter out members with `deleted=True` or `is_bot=True`.
     - For each member, resolve display name: `profile.display_name` or `real_name` or `id`. Strip empty strings.
     - Return the complete `{user_id: display_name}` map.

  5. Replace `resolve_user_names()` implementation:
     - Accept an optional `config: PipelineConfig | None = None` parameter and optional `cache_dir: Path | None = None` parameter (default `Path("config")`).
     - If config is provided, determine `ttl_days = config.slack.user_cache_ttl_days`.
     - Compute `cache_path = cache_dir / "slack_user_cache.json"`.
     - Try to load from disk cache via `_load_user_cache()`.
     - If cache miss: try `_fetch_all_users_batch(client)`.
     - If batch succeeds: save to disk via `_save_user_cache()`, populate `_user_cache` module-level dict, return filtered map for requested `user_ids`.
     - If batch fails (any exception): log warning, fall back to the old per-user `users.info` loop for the requested `user_ids` only.
     - For the returned dict: only include entries for the `user_ids` parameter (callers don't need the full workspace map).
     - Preserve backward compatibility: existing callers that don't pass `config` still work (use default TTL of 7 days).

  6. Update callers in `fetch_slack_items()` and `_resolve_dm_partner()`: pass `config=config` to `resolve_user_names()`.
