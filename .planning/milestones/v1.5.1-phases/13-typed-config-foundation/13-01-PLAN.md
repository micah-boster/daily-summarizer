  <name>Task 2: Update load_config() with error formatting and env var merge</name>
  <files>src/config.py</files>
  <action>
Update `load_config()` to return `PipelineConfig` instead of `dict`. Implement the three-step flow per user decision: load YAML -> merge env vars -> validate via Pydantic.

1. **Rename internal steps:**
   - `_load_yaml(config_path) -> dict` -- reads YAML file, returns raw dict (empty dict if file missing)
   - `_apply_env_overrides(raw: dict) -> None` -- mutates raw dict in place with env var values. Log at DEBUG level per user decision. Keep existing parsing logic (comma-split for calendar IDs).
   - `_validate_config(raw: dict) -> PipelineConfig` -- wraps `PipelineConfig(**raw)` in try/except ValidationError. On error, calls `_format_validation_error()`, prints to stderr, and calls `sys.exit(1)` per user decision.

2. **Error formatter (`_format_validation_error`):**
   - One-line count header: "Config invalid: N error(s)"
   - For each error: field path (dot-separated from `err["loc"]`) + message
   - For `extra_forbidden` errors: use `difflib.get_close_matches(field_name, valid_fields, n=1, cutoff=0.6)` to suggest corrections
   - To get valid fields at a given level, build a helper `_get_valid_fields_at(loc: tuple)` that walks the model tree using `PipelineConfig.model_fields` and nested model classes
   - After each error, if the top-level section has an example in `SECTION_EXAMPLES` dict, show a short valid example snippet
   - Report ALL errors at once per user decision

3. **SECTION_EXAMPLES dict** with short valid YAML snippets for each top-level section (pipeline, calendars, transcripts, synthesis, slack, google_docs, hubspot).

4. **Update `load_config()` signature:** `def load_config(config_path: Path | None = None) -> PipelineConfig`

5. **Add a `make_test_config(**overrides)` factory function** that creates a PipelineConfig with sensible test defaults. This is critical for Plan 02's test migration. It should accept keyword arguments matching top-level section names and merge them into defaults. Example: `make_test_config(slack={"enabled": True, "channels": ["C123"]})`.

**Import additions:** `import difflib`, `import sys`, `import logging`, `from pydantic import BaseModel, ConfigDict, Field, ValidationError, field_validator`
