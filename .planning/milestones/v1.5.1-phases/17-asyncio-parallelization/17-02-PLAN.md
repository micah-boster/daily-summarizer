**Create `src/pipeline_async.py`** with two main async functions:

1. **`async_ingest_all(ctx: PipelineContext) -> tuple`**: Runs all ingest sources concurrently.
   - Import the private ingest functions from pipeline.py: `_ingest_slack`, `_ingest_hubspot`, `_ingest_docs`, `_ingest_notion`. For the calendar chain, create a local async function `_async_calendar_chain(ctx)` that:
     a. Runs the calendar/transcript fetch via `asyncio.to_thread(_fetch_calendar_and_transcripts, ctx)` where `_fetch_calendar_and_transcripts` is a new helper extracted from `_ingest_calendar` that does everything EXCEPT the extraction step (returns categorized, transcripts, unmatched, active_events).
     b. Then runs `extract_all_meetings_async(events_with_transcripts, config, async_client)` for parallel extraction.
     c. Returns (categorized, transcripts, unmatched, extractions).

   Actually -- simpler approach: since `_ingest_calendar` already does everything including extraction, and we want to swap only the extraction part to be async, restructure as follows:

   - Create `_ingest_calendar_async(ctx, async_claude_client)` in pipeline_async.py that mirrors `_ingest_calendar` from pipeline.py but:
     - Wraps the calendar fetch + transcript fetch in `await asyncio.to_thread(...)` using a helper lambda or nested function
     - Replaces the `extract_all_meetings(...)` call with `await extract_all_meetings_async(events_with_transcripts, ctx.config, async_claude_client)`
     - Returns the same tuple: (categorized, transcripts, unmatched, extractions)

   - Use `asyncio.gather()` with `return_exceptions=True` to run all five concurrently:
     ```python
     results = await asyncio.gather(
         asyncio.to_thread(_ingest_slack, ctx),
         asyncio.to_thread(_ingest_hubspot, ctx),
         asyncio.to_thread(_ingest_docs, ctx),
         asyncio.to_thread(_ingest_notion, ctx),
         _ingest_calendar_async(ctx, async_client),
         return_exceptions=True,
     )
     ```
   - For each result, check `isinstance(r, BaseException)` and use empty defaults if so, logging the error.
   - Return (slack_items, hubspot_items, docs_items, notion_items, categorized, transcripts, unmatched, extractions).

2. **`async_pipeline(ctx: PipelineContext) -> None`**: The async entry point.
   - Creates `anthropic.AsyncAnthropic()` client (for async extraction).
   - Records `time.perf_counter()` before ingest.
   - Calls `await async_ingest_all(ctx)` to get all ingest results.
   - Logs ingest wall-clock time: `"Parallel ingest completed in %.1fs"`.
   - Then runs the cross-source dedup, synthesis, commitment extraction, quality tracking, output writing, and Slack notification -- all the same as current `run_pipeline()` from line 240 onward. These are sequential (they depend on ingest results). Copy the logic directly -- it's the same code, just called after the async ingest resolves.
   - Logs total pipeline time.

   - Add `import asyncio` at top.
   - In `run_pipeline()`, replace the sequential ingest block (lines 233-237: the five `_ingest_*` calls) and everything after with a single call: `asyncio.run(async_pipeline(ctx))`. The entire body of `run_pipeline()` becomes just the cache cleanup (lines 221-230) followed by `asyncio.run(async_pipeline(ctx))`.
