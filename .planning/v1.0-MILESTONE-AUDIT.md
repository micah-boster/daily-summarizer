## Integration Issues

### High — Broken User Flow

**Command palette date navigation (NAV-05)**
`CommandPalette.handleSelectDate` → missing bridge → `page.tsx.setSelectedDate`

The handler stores the date in `recentDates` Zustand state and switches to the summaries tab, but `page.tsx` uses local React state for `selectedDate` and does not read from the store. The center panel stays on the previously viewed date.

**Fix:** Either lift `selectedDate` into the Zustand store, or pass a callback from `page.tsx` to `CommandPalette` via the store/context.

### Medium — Dead Feature

**Alias autocomplete (ENT-03)**
`GET /entities/unmatched-mentions` always returns `[]` — the SQL join is unimplemented (explicit TODO in entities.py:148-151). Alias input still works for typed values but never suggests unmatched mentions from the corpus.

### Low — Fragile Contracts

1. Sort value mismatch (`"activity"` vs `"active"`) — works via else fallback
2. `proposal_id=""` sentinel — implicit colon-encoding contract
3. `sidecar: dict` vs `DailySidecar` type — no server-side validation
